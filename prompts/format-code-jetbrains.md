# Format Repository Content (JetBrains)

## Goal

Improve the structure, readability, and consistency of code in a JetBrains-based project (IntelliJ).

## Instructions

### General

- Improve clarity and readability without changing behavior
- Follow standard IntelliJ formatting conventions
- Prefer `.editorconfig` rules over IDE-specific defaults

### Code Quality

- Use clear and descriptive naming
- Break down large or complex functions where appropriate
- Remove dead or redundant code where safe

## Input

{{content}}

## Output

- Cleaned and formatted code
- Brief summary of changes made
