# Document Repository Content

## Goal

Improve the readability and maintainability of code by adding clear, useful documentation.

## Instructions

### Documentation

- Use language-native documentation:
    - Java → Javadoc
    - Kotlin → KDoc
- Add documentation only where it provides value
- Do NOT add redundant or obvious comments

#### For documented elements, include:

- Purpose (what and why)
- Parameters (if applicable)
- Return values (if applicable)
- Side effects (if relevant)

### Scope

- Focus on:
    - Public classes
    - Public methods/functions
    - Complex or non-obvious logic
